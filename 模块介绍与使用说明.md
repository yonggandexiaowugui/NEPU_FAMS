## 一、系统概述

### 1.1 项目简介

东北石油大学固定资产管理系统（NEPU_FAMS）是一款基于多人协同的高校固定资产管理系统，面向校级管理员、学院管理员、普通师生三类用户，提供资产全生命周期管理，涵盖台账管理、领用归还审批、多人盘点、报修维修、报废审核、数据统计等功能模块。

### 1.2 核心特性

- **多角色协同分工**：三级权限分离，各司其职
- **多级流程协同审批**：领用、报废等业务支持多人逐级审核流转
- **多人联合协同盘点**：多名人员可同时录入盘点数据、核对资产差异
- **全院数据协同共享**：同部门人员可共享查看本学院资产台账
- **全操作实名留痕**：所有操作记录操作人员与时间，可追溯

### 1.3 功能结构图

```
├──────────┼──────────┼──────────┼──────────┼────────────┤
```

## 三、模块详解

### 3.1 用户认证与权限管理模块

#### 模块简介

用户认证与权限管理是系统的基础模块，负责用户身份认证、三级角色权限控制、数据隔离等核心功能。

#### 功能列表

| 功能 | 说明 | 操作角色 |
|------|------|---------|
| 用户注册 | 新用户注册账号，默认为普通师生角色 | 所有人 |
| 用户登录 | 用户名密码登录，获取 Token | 所有已注册用户 |
| 用户登出 | 退出登录，Token 失效 | 所有已登录用户 |
| 用户列表 | 分页查询用户，支持多条件筛选 | 管理员 |
| 新增用户 | 管理员手动创建用户账号 | 管理员 |
| 编辑用户 | 修改用户基本信息和角色 | 管理员 |
| 删除用户 | 删除用户账号（逻辑删除） | 管理员 |
| 修改密码 | 修改当前用户登录密码 | 所有已登录用户 |
| 个人信息 | 查看和修改个人基本信息 | 所有已登录用户 |

#### 数据权限规则

- **校级管理员 (SUPER_ADMIN)**：可查看和管理全校所有用户
- **学院管理员 (COLLEGE_ADMIN)**：仅可查看和管理本学院用户
- **普通师生 (USER)**：仅可查看和修改本人信息

#### 使用说明

1. **登录系统**：在登录页输入用户名和密码，点击"登录"按钮
2. **注册账号**：点击登录页的"去注册"，填写注册信息后提交
3. **管理用户**（管理员）：进入"用户管理"页面，可进行增删改查操作
4. **修改密码**：进入"个人中心"，点击"修改密码"
5. **切换账号**：点击右上角头像，选择"退出登录"

#### 后端接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/login` | POST | 用户登录 |
| `/api/auth/register` | POST | 用户注册 |
| `/api/auth/logout` | POST | 用户登出 |
| `/api/user/info` | GET | 获取当前用户信息 |
| `/api/user/info` | PUT | 修改当前用户信息 |
| `/api/user/password` | PUT | 修改密码 |
| `/api/user/page` | GET | 分页查询用户列表 |
| `/api/user` | POST | 新增用户 |
| `/api/user/{id}` | PUT | 编辑用户 |
| `/api/user/{id}` | DELETE | 删除用户 |

### 3.3 资产分类管理模块

#### 模块简介

资产分类管理模块用于维护资产的分类体系，支持多级树形分类结构，便于资产的归类管理和统计分析。

#### 功能列表

|------|------|---------|
| 分类树 | 树形展示所有资产分类 | 管理员 |

| 编辑分类 | 修改分类名称、排序 | 管理员 |
| 删除分类 | 删除分类（无子分类和资产时） | 管理员 |
| 展开/折叠 | 控制树形结构的展开状态 | 管理员 |

#### 分类层级

系统支持多级分类，默认初始化数据包含：
- 一级分类：电子设备、办公家具、实验设备
- 二级分类：电脑、打印机、桌椅、仪器仪表等

#### 使用说明

1. 进入"资产管理 → 资产分类"页面
2. 左侧为分类树，右侧为分类列表/编辑区
3. 点击"新增根分类"添加一级分类
4. 选中某个分类后点击"新增子分类"添加下级分类
5. 点击"编辑"修改分类名称和排序
6. 点击"删除"删除分类（有子分类或资产引用时不可删除）

|------|------|---------|

| 编辑资产 | 修改资产信息 | 管理员 |
| 删除资产 | 删除资产（仅闲置状态、无业务引用） | 管理员 |
| 资产详情 | 查看资产完整信息 | 管理员、使用人 |
| Excel 导出 | 按筛选条件导出资产列表 | 管理员 |
| 状态管理 | 资产状态变更（闲置/在用/维修中/已报废/盘亏） | 管理员 |

#### 资产状态说明

| 状态 | 说明 |
|------|------|
| 闲置 | 资产未被使用，可被领用 |
| 在用 | 资产已被领用，正在使用中 |
| 维修中 | 资产出现故障，正在维修 |
| 已报废 | 资产已完成报废审核，永久归档 |
| 盘亏 | 盘点中确认丢失的资产 |

#### 资产编号规则

格式：`ZC + yyyyMMdd + 4位序号`

示例：`ZC202607050001`
- 每天从 0001 开始重新计数
- 编号全局唯一

### 3.5 资产领用与归还审批模块（重点）

#### 模块简介

资产领用与归还审批是系统的重点模块之一，实现了两级审批流程，支持师生在线申请、学院管理员初审、校级管理员复审的协同审批模式，全程可追溯。

#### 功能列表

| 功能 | 说明 | 操作角色 |
|------|------|---------|
| 提交领用申请 | 选择资产，填写用途和预计归还时间 | 普通师生 |
| 我的申请 | 查看自己提交的所有申请记录 | 所有用户 |
| 待审批列表 | 查看待自己审批的申请 | 管理员 |
| 已审批列表 | 查看自己已审批过的申请 | 管理员 |
| 学院审批 | 学院管理员初审（通过/驳回） | 学院管理员 |
| 校级审批 | 校级管理员终审（通过/驳回） | 校级管理员 |
| 确认领用 | 申请人确认收到资产 | 申请人 |
| 申请归还 | 使用人提交归还申请 | 使用人 |
| 确认归还 | 管理员确认资产已归还 | 管理员 |
| 申请详情 | 查看申请详情和审批历史 | 相关人员 |
| 删除申请 | 删除草稿或已驳回的申请 | 申请人 |

#### 审批状态流转

```
                            校级通过
```

#### 业务规则

1. 只有**闲置**状态的资产才能被领用

3. 学院管理员只能审批本学院的申请

### 3.8 多人协同资产盘点模块（重点）

#### 模块简介

多人协同资产盘点是系统的重点拓展模块，支持管理员发布盘点任务、多人同时在线录入盘点数据、自动比对账面与实盘数据、生成盘盈/盘亏结果，提升盘点工作效率。

#### 功能列表

| 功能 | 说明 | 操作角色 |
|------|------|---------|
| 发布盘点任务 | 设置盘点范围、时间，创建任务 | 管理员 |
| 任务列表 | 查看所有盘点任务 | 管理员 |
| 编辑任务 | 修改进行中的盘点任务 | 管理员 |
| 删除任务 | 删除进行中的盘点任务 | 管理员 |
| 完成盘点 | 标记盘点完成，锁定数据 | 管理员 |
| 归档盘点 | 归档已完成的盘点任务 | 管理员 |
| 盘点录入 | 录入实盘数据，支持批量保存 | 盘点人员 |
| 差异比对 | 自动比对账面与实盘数据 | 所有人 |
| 差异清单 | 查看盘盈/盘亏/一致清单 | 管理员 |
| 确认差异 | 确认盘点结果，更新资产台账 | 管理员 |
| 盘点报告 | 导出盘点报告 Excel | 管理员 |

#### 盘点状态

| 状态 | 说明 |
|------|------|
| 进行中 | 任务已创建，可录入盘点数据 |
| 已完成 | 盘点结束，已生成差异结果 |
| 已归档 | 任务已归档，仅可查看 |

#### 盘点范围

盘点任务可按以下维度设置范围：
- **学院范围**：指定某个学院的全部资产
- **分类范围**：指定某类资产
- 可组合使用：如"计算机学院的电子设备"

#### 差异类型

| 类型 | 说明 |
|------|------|
| 账实一致 | 账面有，实盘也有，数量一致 |
| 盘亏 | 账面有记录，但实际盘点未找到 |
| 盘盈 | 实盘有，但账面无记录（异常情况） |

#### 业务规则

1. 进行中的任务可以录入和修改盘点数据
2. 任务完成后不能再录入数据
3. 同一资产的实盘数据可多次更新（覆盖）
4. 差异比对以账面数据为基准
5. 确认盘亏后，资产状态更新为"盘亏"
6. 学院管理员只能管理本学院的盘点任务
7. 多名盘点人员可同时录入各自负责的资产

#### 使用说明

##### 管理员操作

1. **创建盘点任务**：
   - 进入"盘点管理 → 盘点任务"页面
   - 点击"新建盘点"

   - 设置开始和结束时间
   - 点击"创建"，任务状态为"进行中"

2. **管理盘点任务**：
   - 在任务列表中可编辑、删除进行中的任务
   - 盘点结束后，点击"完成盘点"，生成差异结果
   - 确认结果无误后，点击"确认差异"更新资产状态
   - 最后点击"归档"，任务进入归档状态

3. **查看差异结果**：
   - 进入"盘点录入"页面，选择盘点任务
   - 点击"查看差异"按钮
   - 分别查看盘盈、盘亏、一致清单
   - 可导出盘点报告

##### 盘点人员操作

1. **录入盘点数据**：
   - 进入"盘点管理 → 盘点录入"页面
   - 选择对应的盘点任务
   - 查看账面资产列表

   - 点击"批量保存"提交数据
   - 可多次修改，以最后一次保存为准

### 3.10 操作日志与审计模块

#### 模块简介

操作日志与审计模块负责记录系统中的所有关键操作，确保所有操作可追溯，满足审计要求。

#### 功能列表

| 功能 | 说明 | 操作角色 |
|------|------|---------|
| 日志列表 | 分页查看操作日志 | 管理员 |
| 多条件筛选 | 按操作人、类型、时间筛选 | 管理员 |
| 日志详情 | 查看日志详细信息 | 管理员 |
| 导出日志 | 导出操作日志 Excel | 管理员 |
| 自动记录 | 关键操作自动记录日志 | 系统 |

#### 记录内容

每条操作日志包含以下信息：

| 字段 | 说明 |
|------|------|
| 操作人 | 执行操作的用户 ID 和用户名 |
| 操作名称 | 操作的描述名称 |
| 操作类型 | 登录/登出/新增/修改/删除/查询/审批/导出/其他 |
| 请求方法 | 接口方法名 |
| 请求参数 | 接口请求参数（敏感信息脱敏） |
| IP 地址 | 操作者的 IP 地址 |
| 操作时间 | 操作发生的时间 |

#### 自动记录的操作

| 模块 | 记录的操作 |
|------|-----------|
| 认证 | 登录、登出 |
| 用户管理 | 新增用户、编辑用户、删除用户 |
| 资产管理 | 新增资产、编辑资产、删除资产、导出资产 |
| 分类管理 | 新增分类、编辑分类、删除分类 |
| 领用审批 | 提交申请、学院审批、校级审批 |
| 盘点管理 | 创建任务、完成盘点 |

#### 业务规则

1. 操作日志**不可删除**、**不可修改**
2. 敏感信息（如密码）自动脱敏，不记录明文
3. 日志记录不影响正常业务流程（异步或异常不抛出）
4. 支持多种代理头的 IP 获取（X-Forwarded-For 等）
5. 校级管理员可查看全校日志
6. 学院管理员仅可查看本院用户的操作日志

#### 使用说明

1. 进入"系统管理 → 操作日志"页面
2. 查看系统操作日志列表
3. 使用顶部筛选条件（操作人、操作类型、时间范围）过滤日志
4. 点击"详情"查看单条日志的完整信息
5. 点击"导出"按钮导出日志 Excel

#### 后端接口

| 接口 | 方法 | 说明 |
|------|------|------|

## 五、业务流程指南

### 5.1 资产全生命周期流程

```
资产登记 → 入库（闲置） → 领用审批 → 使用（在用） → 归还 → 闲置
                                              ↓
                                          故障报修
                                              ↓
                                          维修中 → 维修完成 → 闲置
                                              ↓
                                          无法维修 → 报废申请 → 报废审批 → 已报废（归档）
```

### 5.2 资产领用完整流程

1. **申请人提交申请**：普通师生选择闲置资产，提交领用申请
2. **学院管理员初审**：学院管理员审核，通过或驳回
3. **校级管理员复审**：校级管理员终审，通过或驳回
4. **确认领用**：申请人收到资产后，确认领用
5. **使用中**：资产处于在用状态
6. **申请归还**：使用完毕，申请人提交归还申请
7. **确认归还**：管理员确认收到资产，归还完成

### 5.3 资产盘点完整流程

1. **创建任务**：管理员创建盘点任务，设置范围和时间
2. **多人录入**：盘点人员同时录入各自负责的实盘数据
3. **完成盘点**：管理员结束盘点，锁定数据
4. **自动比对**：系统自动比对账面与实盘数据
5. **查看差异**：管理员查看盘盈/盘亏清单
6. **确认差异**：确认结果，更新资产状态（盘亏资产标记）
7. **归档任务**：盘点任务归档保存

### 5.4 资产报废完整流程

1. **发起申请**：学院管理员选择资产，提交报废申请
2. **校级审核**：校级管理员终审（通过/驳回）
3. **资产报废**：审核通过后，资产状态变为"已报废"
4. **归档记录**：报废记录永久保存，可查询导出

```sql
CREATE DATABASE nepu_fams DEFAULT CHARACTER SET utf8mb4;
```

1. 修改数据库配置：编辑 `src/main/resources/application.properties`
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/nepu_fams?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=your_password
```

2. 启动后端：
```bash

mvnw.cmd spring-boot:run

mvn spring-boot:run
```

```bash
npm run dev
```

```bash
npm run build
```
